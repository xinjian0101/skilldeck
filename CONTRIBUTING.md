# Contributing to SkillDeck

Thank you for contributing to SkillDeck.

## Before starting

1. Read `README.md`, `ABOUT.md`, and `ROADMAP.md`.
2. Search existing Issues and pull requests.
3. Open an Issue before making a large architectural or data-format change.
4. Keep documentation aligned with implemented behavior.

## Contribution areas

- Interface and accessibility improvements.
- Data-model and validation work.
- Import and export compatibility.
- Tests, fixtures, examples, and documentation.
- Performance and reliability improvements.

## Commits

Use focused Conventional Commit messages such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

## Pull requests

Describe the problem, intended behavior, non-goals, validation steps, compatibility impact, and documentation changes. Keep unrelated changes in separate pull requests.

Contributors are responsible for correctness, licensing, and review of all submitted material.

## Conduct

Be specific, technical, and respectful. Do not include private information or material that you do not have permission to publish.
